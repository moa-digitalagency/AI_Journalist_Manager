J’ai besoin de développer une webapp intégrant un agent IA capable de créer des « journalistes IA » pour la production de résumés d’actualités.

L’administrateur se connecte, crée un journaliste en le liant avec un bot Telegram, puis charge les sources souhaitées (liens de sites web pour la veille médiatique, comptes X/Twitter et flux RSS). Le journaliste IA collecte automatiquement, toutes les 24 heures, les nouvelles publications des sources, les stocke dans sa base de données avec leurs métadonnées, puis génère un résumé. Ce résumé est ensuite envoyé automatiquement chaque matin sur la conversation Telegram (dans un premier temps) reliée au journaliste concerné.

Le journaliste IA produit un résumé court et concis, en version texte et audio, grâce à l’API Eleven Labs.
Chaque journaliste IA dispose d’une personnalité, d’un style d’écriture, d’une orthographe spécifique et d’un ton de réponse configurables. Ces paramètres déterminent le style du résumé, ainsi que la manière dont le journaliste interagit avec les utilisateurs. En effet, il peut discuter en langage naturel avec toute personne connectée à lui.

Les utilisateurs peuvent lui demander une information précise : le journaliste analyse alors sa base de données, détecte les mots-clés correspondants et renvoie l’information pertinente en mentionnant toujours la source.

Chaque nouvelle personne qui engage une conversation est considérée comme un abonné potentiel.
Depuis l’interface administrateur, l’accès aux fonctionnalités du bot peut être validé et limité dans le temps : par exemple, un essai d’une semaine pendant laquelle l’utilisateur reçoit les résumés quotidiens et peut interagir avec le journaliste. Une fois cette période expirée, tous les services sont suspendus jusqu’à réactivation.

Le panel administrateur sera complet pour permettre une meilleure gestion des journalistes, avec un menu latéral comprenant :
- Un tableau de bord avec les statistiques
- Un menu pour gérer et configurer les bots
- Un menu pour gérer les abonnés qui commencent une conversation avec un bot, valider ou assigner les forfaits
- Un menu pour gérer les forfaits
- Un menu de statistiques plus détaillé
- Un menu pour consulter les logs d’activités et de sécurité
- Un menu pour gérer les utilisateurs de la plateforme, créer d’autres administrateurs avec gestion des rôles
- Un menu de paramètres pour gérer la configuration de la plateforme, le SEO et d’autres réglages internes

Toutes les informations critiques, y compris les identifiants superadmin, doivent être stockées dans le fichier .env.
Le design doit être professionnel, moderne, stylé et très épuré, avec des tons bleu foncé.

Stack technique :
- Backend : Python
- Frontend : HTML, CSS, Tailwind, JavaScript
- Base de données : PostgreSQL
- IA : Gemini
- Text-to-Speech : Eleven Labs (voix IA personnalisée)
- Interface de communication : Telegram (un bot par journaliste, utilisant le token généré du bot)

Structure des dossiers :
Les fichiers doivent être regroupés en dossiers selon leur rôle (routes, models, services, security, utils, log, lang, etc.), chacun contenant les fichiers qui s’y réfèrent.
